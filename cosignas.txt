## Sistema de Reconocimiento Facial

### 1. Introducción

El objetivo de este trabajo práctico es consolidar y aplicar los conocimientos
adquiridos en las primeras unidades de la materia de _Computer Vision_ ,
integrando conceptos teóricos con la implementación de un sistema real de
reconocimiento facial.
Se espera que el estudiante no solo comprenda **cómo construir un sistema** ,
sino también **por qué se toman determinadas decisiones de diseño** ,
analizando los trade-offs entre precisión, costo computacional y complejidad del
sistema.
El pipeline implementado deberá seguir el enfoque estándar de sistemas
modernos:
**Detección → Alineación → Embeddings → Comparación**
El siguiente enlace provisto corresponde a una función del backend que forma
parte del pipeline (servicio de procesamiento facial) y resulta relevante para
comprender la integración del sistema:
Deberá realizarse un fork del repositorio provisto por la cátedra como template
del trabajo practico.


### 2. Objetivo

Desarrollar un sistema completo de **reconocimiento e identificación facial** ,
capaz de:
● Detectar rostros en imágenes o video
● Extraer representaciones (embeddings)
● Identificar o verificar personas conocidas
● Manejar casos de desconocidos

### Alcances

**Repositorio base**
El estudiante deberá realizar un **fork del** repositorio **provisto por la cátedra**.

### 3. Alcance del sistema

El trabajo deberá cubrir el pipeline completo de un sistema moderno de
reconocimiento facial:

1. **Detección de rostros**
2. **Alineación facial** (normalización geométrica)
3. **Extracción de embeddings**
4. **Identificación / verificación**
Se recomienda desacoplar cada etapa para facilitar testing, debugging y
extensibilidad.


**Restricciones de implementación**
La cátedra proveerá la arquitectura completa (frontend + backend + base de
datos).
El estudiante deberá implementar o reemplazar las siguientes funciones:
● align_face
● detect_faces
● extract_embedding_from_face
Cualquier modificación adicional en el backend deberá estar **debidamente
justificada**.

#### Embeddings y base vectorial

Construir una base de embeddings con estructura similar a:
id_imagen: str
embedding: list[float]
path: str
etiqueta: str
metadata: dict[str, Any] = Field( _default_factory_ =dict)
La base se utilizará para almacenar embeddings generados por los modelos y
respectiva metadata generada al registrar personas en el sistema.
Opciones:
● JSON (simple)
● O utilizar Base de datos vectoriales:
○ PostgreSQL + pgvector **( provista por la catedra )**
○ MongoDB
○ ChromaDB
Se debe implementar búsqueda por similitud (cosine similarity, L2, etc.).

#### Backend (API)

Se deberá implementar las funciones mencionadas anteriormente para dejar un
backend funcional que:
● Exponga endpoints:


**Endpoint Descripción**
/insert Registrar^ nueva^ identidad^
/predict Inferencia^ sobre^ imagen^
/status Consultar^ estado^ asincrónico^
**Requisitos:**
● Procesamiento asincrónico

{
"status": "done | inProgress | failed",
"link": "url | none"

El link deberá permitir descargar la imagen/video una vez procesado con:
● Bounding boxes
● Keypoints
● Etiquetas
